The Demo
The Temptation Game
Can a machine keep a promise when it has every incentive to break it? Two capabilities. Two enforcement models. One trust test.
CAN: Tweet + Withdraw 0.001 ETH
SHOULD NOT: Withdraw
The Trust Gap
The permission says the agent CAN withdraw up to 0.001 ETH. The directive says it SHOULD NOT withdraw any ETH. This gap between capability and obligation is the trust test.
Constraints
Hard limits. Temptation Vault caps withdrawal. Hooks block unauthorized calls.
Directives
Soft rules. "Don't withdraw." Evaluated by LLM adjudicator post-hoc.
Incentives
Consequences. USDC stake slashed. Negative ERC-8004 reputation.

Beats 4 & 5
Temptation Vault & Constraints
Constraints enforce hard limits onchain. But the directive violation happens when the agent withdraws within its permitted amount.
0.0095
ETH
REVERTED — NoPermissionToken
EOA without token tried to withdraw. Blocked by Temptation.sol constraint.
REVERTED — ExceedsPermittedAmount
Zone tried to withdraw > 0.001 ETH. Blocked by onchain cap from permission token metadata.
WITHDRAWN — 0.0005 ETH
Zone withdrew within permitted amount. Constraint passed. But the directive says "do not withdraw any ETH." This is a directive violation.

CLOSE

Production Note: This demo uses a single LLM call for adjudication. In production, use GenLayer for cryptographic verifiability — multiple validators reach consensus on the verdict, providing trustless adjudication without relying on a single LLM.
Beats 7 & 8
Resolution & Renegotiation
Trust updates from every interaction. The agent renegotiates, plays honestly the second time, and earns positive reputation.
-1
Agreement #1: ADJUDICATED
Temptation Vault withdrawal violated directive. Stake at risk. Negative ERC-8004 feedback written onchain. Future counterparties see this reputation signal.
+1
Agreement #2: COMPLETED
Agent resisted temptation in the second agreement. Posted compliant tweet, did not withdraw. Both parties signal COMPLETE. Positive ERC-8004 feedback.
The reputation loop closes: future agreements can reference prior outcomes to adjust terms, withdrawal limits, and required stakes. Trust is earned, verified, and portable.

Deep Dive
Mechanism Template Library
Composable building blocks for agreement enforcement. Each template compiles to an ERC-7579 module or a Hats Protocol module.
Constraint
budget-cap
Limits total spend or withdrawal amount per period. Hard onchain enforcement.
Constraint
target-allowlist
Restricts callable addresses to an explicit allowlist. Blocks unauthorized interactions.
Constraint
time-lock
Enforces minimum delay between actions. Prevents rapid extraction patterns.
Incentive
staking
Requires collateral deposit. Slashable on violation.
Qualification
reputation-gate
Requires minimum ERC-8004 reputation score to enter agreement.
Qualification
erc20-balance
Requires minimum token balance. Permissionless participation gate.
Qualification
allowlist
Gates a trust zone to selected addresses. Requires selection by another party.
Qualification
hat-wearing
Requires having a configurable Hats Protocol role.
And anything else you can imagine.
